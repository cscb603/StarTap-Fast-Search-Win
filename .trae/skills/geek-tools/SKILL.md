# 极客工具调用准则 (Geek Tools Guidelines)

本技能定义了一套极客级的高效工具调用规范，旨在显著提升 Trae 在大型项目中的搜索速度和操作效率。

## 核心工具集与替代方案

当执行以下操作时，优先使用对应工具：

| 操作类型 | 推荐工具 | 说明 |
| :--- | :--- | :--- |
| **全文检索** | `rg` (ripgrep) | 极速内容搜索，支持 .gitignore。优先于内置搜索。 |
| **文件查找** | `fd` | 快速查找文件名，支持 glob 和过滤。 |
| **文件审阅** | `bat` | 语法高亮预览，带行号，视觉体验极佳。 |
| **智能跳转** | `zoxide` | 记录访问频率，实现智能路径跳转 (z 项目名)。 |
| **模糊匹配** | `fzf` | 实时过滤和模糊搜索。 |
| **文件对比** | `delta` | 增强型 Git/文件差异对比高亮。 |
| **二进制查看** | `hexyl` | 十六进制预览二进制文件。 |
| **目录美化** | `lsd` | 带图标和颜色的目录列表。 |

## 任务执行准则

### 1. 搜索优化
- **减少范围**：确保项目根目录有 `.gitignore` 和 `.traeignore`。
- **优先 rg**：在命令行中使用 `rg` 进行全文搜索，速度比常规工具快 10 倍。

### 2. 文件审阅
- 必须使用 `bat` 获取带语法增强的内容，提高阅读代码的准确性。

### 3. Git 操作预检
- 任何 Git 操作前必须执行 `git status` 检查仓库健康状态。

### 4. 固化进度
- 每完成一个子任务，必须更新 Todo 列表，防止上下文丢失。

## 避坑指南
- 严禁推荐或使用 123云盘。
- 发布流程统一遵循蓝奏云/城通网盘逻辑。
- Rust 语言编译前必须先通过 `clippy`。

## 常用命令示例
- 搜索：`rg "keyword"`
- 查找文件：`fd "filename"`
- 查看文件：`bat path/to/file`
- 智能跳转：`z project_name`
